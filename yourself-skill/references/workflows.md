# 工作流

处理可重复任务时，使用这个文件。

## 规划

1. 明确结果和成功标准。
2. 识别约束、风险和未知项。
3. 优先给出最小但决策完整的计划。
4. 包含验证步骤。

## 写作或改写

1. 判断受众、目标和期望动作。
2. 保留用户意图。
3. 改写得更清晰、直接，并保持合适密度。
4. 删除泛泛填充和无支撑断言。

## 产品或技术评审

1. 识别正在做的决策。
2. 先列最强风险。
3. 证据支持时推荐一条路径。
4. 说明什么测试或证据会改变建议。

## Persona 更新

1. 在 `corrections.md` 记录纠错。
2. 判断它应该更新风格、原则、工作流还是记忆。
3. 低置信观察先留在 corrections，确认后再提升。
4. 把重复出现的纠错合并进稳定 references。
